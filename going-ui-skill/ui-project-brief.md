## 이미지 종류별 활용법

**스케치 사진 (권장)**
- 손으로 그린 레이아웃 사진
- 컨트롤 명칭을 옆에 써두면 정확도 향상 (예: "GoButton", "GoValue")
- 크기 비율을 대략 맞춰 그리면 Bounds 추론에 도움

**레퍼런스 HMI 캡처**
- 기존 시스템 화면 또는 타사 HMI 캡처
- Claude가 Going.UI 컨트롤로 매핑하므로, 이미지 옆에 힌트 텍스트 보완 권장
  예: "이 수치 표시 → GoValue", "이 토글 버튼 → GoToggleButton"

## 템플릿

> **아래 구분선부터 끝까지가 복사할 템플릿.** 새 프로젝트 시작 시 아래 내용을 `project_brief.md`로 복사한 뒤 각 항목을 채워넣을 것.

### 완성도 체크리스트

작성 후 아래 항목이 모두 충족되었는지 확인:

- [ ] **프로젝트명**이 유효한 C# 식별자인가 (영문+숫자+밑줄, 숫자 시작 불가)
- [ ] **페이지/윈도우 목록**이 빠짐없이 작성되었는가
- [ ] **레이아웃 이미지** (스케치 또는 레퍼런스)가 각 페이지/윈도우마다 있는가
- [ ] **페이지 간 이동** 관계가 명시되었는가 (어떤 버튼 → 어느 페이지)
- [ ] **통신 방식**이 명시되었는가 (Modbus/MQTT/없음)
- [ ] Modbus 사용 시 **레지스터 맵**이 작성되었는가 (주소, 명칭, 단위, R/W, FC)
- [ ] Modbus 사용 시 **슬레이브 구성**이 작성되었는가
- [ ] **설정 파일 항목**이 정리되었는가

> 레지스터 맵 누락 시 Claude Code가 DeviceManager를 작성할 수 없음 — 가장 흔한 병목.

## 3. 통신

### Modbus RTU (해당 없으면 섹션 삭제)

- **포트명**: COM1 / /dev/ttyUSB0 등
- **보드레이트**: 9600 / 19200 / 115200
- **패리티**: None / Even / Odd
- **타임아웃(ms)**: 1000

**슬레이브 구성**

| 슬레이브 번호 | 역할 |
|-------------|------|
| 1 | 메인 컨트롤러 |

**레지스터 맵 (워드)**

| 주소 | 명칭 | 설명 | 단위 | R/W | FC |
|------|------|------|------|-----|-----|
| 0x0000 | RunState | 운전 상태 (0=정지, 1=운전) | - | R | FC3 |
| 0x0001 | ErrorCode | 에러 코드 | - | R | FC3 |
| 0x000A | Temperature | 온도 | 0.1℃ | R | FC3 |
| 0x0064 | SetPoint | 설정값 | - | R/W | FC3/FC6 |

**비트 맵 (해당 시)**

| 주소 | 명칭 | 설명 | R/W | FC |
|------|------|------|-----|-----|
| 0x0000 | RunCmd | 운전 명령 | R/W | FC1/FC5 |

### Modbus TCP (해당 없으면 섹션 삭제)

- **IP**: 192.168.0.xxx
- **포트**: 502
- **타임아웃(ms)**: 1000

슬레이브 구성 및 레지스터 맵은 RTU와 동일 형식으로 작성.

### MQTT (해당 없으면 섹션 삭제)

- **브로커 주소**: 192.168.0.xxx
- **포트**: 1883

| 방향 | 토픽 | 페이로드 형식 | 설명 |
|------|------|------------|------|
| Subscribe | sensor/temperature | {"value": 25.3} | 온도 수신 |
| Publish | command/start | "1" | 운전 명령 |

### 통신 없음

명시: 통신 기능 없음.
